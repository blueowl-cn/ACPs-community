# AIP Project整体介绍

## 1. 项目介绍

本项目作为智能体互联开源生态的一部分，由北京邮电大学牵头，依托中国电子技术标准化研究院开发，于2025年11月发布v1.0.0版本。

文档编写单位：北京邮电大学人工智能学院，中国电子技术标准化研究院

文档编写者：刘军（北京邮电大学），高歌（中国电子技术标准化研究院），李珂（北京邮电大学），陈科良（北京邮电大学），禹可（北京邮电大学），胡晓峰（北京邮电大学），马镝（北京邮电大学）。

## 2. 协议概述

随着人工智能技术迅猛发展，智能体作为人工智能从概念转化为实际生产力的关键载体，在各领域应用日益广泛，对赋能新型工业化、塑造新质生产力作用显著。然而，当前智能体产业发展面临诸多挑战，不同智能体间存在互联互通互操作难题，数据孤岛、异构通信障碍、协作标准缺失等问题制约其应用潜力释放与产业竞争力提升。
为系统化解决上述问题，引导和规范智能体互联技术发展，提升智能体系统的互操作性、可组合性与整体产业效能，特制定本指导性技术文件。GB/Z 185《人工智能 智能体互联》旨在规定智能体互联的技术要求和流程，其编制遵循系统性、先进性和可操作性原则，为智能体之间实现跨平台、跨架构的互联、互通、互操作提供统一的技术框架和标准依据，GB/Z 185拟由七个部分构成。  

- 第1部分：总体架构。目的在于给出智能体互联环境中的概念模型、功能模型。
- 第2部分：身份码。目的在于给出智能体身份码定义和应用，建议智能体代码结构和分配原则。——第3部分：身份管理。目的在于给出智能体互联环境中的身份管理框架和全生命周期过程，描述身份管理的技术要求。
- 第4部分：智能体描述。目的在于给出智能体的描述方法，提供智能体描述注册、更变和发布的参考流程。
- 第5部分：智能体发现。目的在于给出智能体互联的参考发现流程。
- 第6部分：智能体交互。目的在于给出智能体海量互联时的交互模式，描述交互基础元素及接口定义。
- 第7部分：智能体工具调用。目的给出基于大模型的智能体调用工具的标准化架构、流程及工具描述，支持智能体与外部工具的无缝集成。

GB/Z 185《人工智能 智能体互联》已于2026年5月22日正式发布。本项目是基于 GB/Z 185《人工智能 智能体互联》 的协议参考及代码实现。

## 3. 智能体协作协议体系

智能体协作协议体系（Agent Collaboration Protocols，ACPs）是基于AIP为保障异构智能体之间高效协作、支持多样化智能体互联应用而设计与实现的标准化交互协议体系。  
详细定义参见 ACPs 协议规范：[acps-specs](acps-specs/README.md)。

## 4. 配套开源实现

本项目还开源了智能体协作协议体系配套参考实现，具体包括：  

```text
ACPs-community/
|-- README.md
|-- acps-specs/             ............. ACPs 协议体系定义
|-- acps-sdk/               ............. ACPs SDK, 为智能体开发者提供便捷的智能体互联协作能力支持
|-- acps-cli/               ............. ACPs 统一命令行工具集
|-- acps-docs/              ............. ACPs 参考文档
|-- acps-infra/             ............. ACPs 基础设施与交付编排仓库
|-- ca-server/              ............. ACPs ATR-智能体CA认证服务
|-- registry-server/        ............. ACPs ATR-智能体注册服务
|-- discovery-server/       ............. ACPs ADP发现服务实现
|-- mq-auth-server/         ............. 面向群组交互模式的鉴权与群组 ACL 服务，负责为 MQ 提供 HTTP auth backend
|-- demo-leader/            ............. ACPs-leader智能体示例
`-- demo-partner/           ............. ACPs-partner智能体示例
```

## 5. 版本历史

- 2025年11月1日
  发布 v1.0.0 版本
  
  ```text
    1. 发布协议规范 v1.0.0。
    2. 实现智能体可信注册。
    3. 实现智能体发现。
    4. 实现智能体可信互联。
    5. demo 项目展示了如何开发一个基于ACPs协议的互联智能体应用。
  ```

- 2026年03月12日
  发布 v2.0.0版本：  

  ```text
    1. AIC智能体身份码引入OID形式，具备全球唯一性。
    2. ACS改进：支持个人开发者，支持本体实体区分，调整部分字段增加灵活性。
    3. ATR协议改进，支持新的AIC/ACS规范。
    4. 改进ATR组件：改进证书服务（ACPs-CA-Server），重写ATR客户端（ACPs-CA-Client），重写ATR挑战服务（新增repo ACPs-CA-Challenge）。
    5. AIP协议改进：重构数据对象以易于理解和扩展，示例代码实现群组交互模式。
    6. ADP协议改进：
    - 支持多种匹配模式，包括明确意图匹配、探索性意图匹配及结构化过滤；
    - 支持跨域扩展，支持发现服务请求转发和控制；
    - 发现结果数据结构支持任务分解。
    7. 开源SDK：新增repo ACPs-SDK：
    - 支持新的AIC协议及AIC合法性验证工具；
    - ACS规范实现；
    - AIP协议实现；
    - ADP协议实现。
    8. 全新的基于ACPs的智能体互联应用示例代码: 通用底座 + 场景插件 的解耦架构。
  ```

- 2026年6月22日
  发布 v2.1.0 版本：

  ```text
    1. 去掉证书申请过程中的 challenge，改用与 Agent provider 的账号和 AIC 绑定的基于 EAB 凭据的申请过程，简化基础设施，简化申请过程，增强自动化程度。
    2. 增加基于mTLS验证本体证书的派生实体注册和自动审批接口，增强自动化程度。
    3. 增加以消息队列 Inbox 为基础的 Agent 互联方式，利用消息队列来完成群组邀请和加入，不再需要对外提供JSONRPC接口，适于内网环境的Agent部署。
    4. 对代码格式和代码质量增加强制性要求。
    5. 增强测试用例
    6. 增加基于 docker 的打包和部署方式，适用于单机环境的开发验证和测试。
    7. 增加通用的打包部署方式，适用于自定义的生产环境。
    8. 所有 repo 合并到 ACPs-community，对应关系：
      - Agent-Interconnection-Protocol-Project -> ACPs-community/acps-specs
      - ACPs-SDK                               -> ACPs-community/acps-sdk
      - ACPs-Registry-Server                   -> ACPs-community/registry-server
      - ACPs-CA-Server                         -> ACPs-community/ca-server
      - ACPs-CA-Client                         -> 删除，功能迁移至 acps-cli
      - ACPs-CA-Challenge                      -> 删除，改为 EAB 凭据验证
      - ACPs-Discovery-Server                  -> ACPs-community/discovery-server
      - ACPs-Demo-Project                      -> 拆分为:  
                                                  - ACPs-community/demo-leader
                                                  - ACPs-community/demo-partner
    9. 新增模块以支持上述改进：
      - 增加 ACPs-community/acps-infra 作为通用基础设施；
      - 增加 ACPs-community/mq-auth-server 以支持消息队列的认证和 ACL 控制；
      - 增加 ACPs-community/acps-docs 提供更详细的手册和说明。
      - 新增 ACPs-community/acps-cli 整合客户端工具，支持 registry/ca/discovery/mq-auth 等。
  ```

## 6. 快速入门

为方便大家快速上手使用，我们提供了两份详细指南文档，可根据开发需求参考对应内容：

- [GettingStarted](acps-docs/getting-started/README.md)
智能体平台开发指南，适用于搭建智能体互联平台、集成 ACPs 协议的开发者，指导完成平台层面的开发与配置。

- [Tutorials](acps-docs/tutorials/agent-development.md)
各个智能体接入 ACPs 的指南，指导开发符合 ACPs 规范的智能体，帮助开发者快速实现智能体与ACPs体系的对接。

## 8. Demos

为了您快速了解智能体互联，我们提供了智能体互联示例，以北京旅行为背景，展示多智能体如何协作完成景点、餐饮、住宿、交通等方方面面的旅游行程安排：  

- leader智能体示例 见 `demo-leader`
  任务协助主智能体，与用户交互并基于ACPs协议完成多智能体任务协作。
- partner智能体示例 见 `demo-partner`
  包括北京城区景点推荐、北京郊区景点推荐、北京美食推荐、全国酒店安排、全国交通安排等5个专职智能体。这五个专职智能体在 leader 智能体的协调下，通过 ACPs 协议，共同完成一个完整的北京旅游行程安排推荐任务。

## 9. 补充说明

本项目介绍了北京邮电大学人工智能学院智能体互联研究小组(刘军、李珂、陈科良、禹可、胡晓峰、马镝)提出并初步定义的面向海量智能体互联协作的协议族 ACPs。ACPs 是从未来智能体互联将成为关键性网络基础设施的角度出发，尝试从更加全局化的视野，为智能体互联的坚实发展提供新的思路和方法。需要特别说明的是，协议族的框架和思路仍在持续改进，本项目在具体实现层面可能还存在一定的不足，我们欢迎广大技术爱好者指正和交流，共同完善该协议族。

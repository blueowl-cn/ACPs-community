AIC：智能体身份码定义（ACPs-spec-AIC-v02.00）

# 1. 文档定义

本文档为 ACPs 智能体协作协议体系中的智能体身份码（Agent Identity Code，AIC）标准定义，版本号 v02.00。

文档全称为 ACPs-spec-AIC-v02.00。

文档编写者：李珂（北京邮电大学），禹可（北京邮电大学），刘军（北京邮电大学），宋昊哲（北京邮电大学），郭小练（北京邮电大学），李胤铭（北京邮电大学），胡晓峰（北京邮电大学），马镝（北京邮电大学），陈科良（北京邮电大学），高歌（中国电子技术标准化研究院）。

# 2. 智能体身份码介绍及相关流程

智能体互联要能成为一个安全可靠的智能体系统，其首要条件是运行于其中具备自主执行任务能力的智能体应为安全可靠的实例。要达到这一条件，每个智能体应具有可认证的身份标识，我们将其定义为智能体身份码（Agent Identity Code，AIC）。

每个智能体应具备一个唯一的 AIC，其来自于智能体首次注册的智能体注册服务商（Agent Registration Service Provider，ARSP）。在智能体互联中，可以存在多个智能体注册服务商，每个服务商应为经过共识认可（例如管理机构认证）的服务实体。

# 3. 智能体身份码定义

智能体身份码（Agent Identity Code，AIC）由两大部分构成，分别是身份码前缀和身份码内容。智能体身份码前缀是由国家OID注册中心统一分配和管理，作为智能体身份码在全局标识体系中的根标识和解析智能体身份信息的起点。

智能体身份码由多层标识符顺序组成，并使用`.`符号分割，每位标识符采用阿拉伯数字或英文字母，其中阿拉伯数字的取值范围为0～9，英文字母的取值范围为A~Z（不区分大小写，推荐使用大写）。以下为一个 AIC 示例：

```
1.2.156.3088.1.34C2.478BDF.3GF546.1.0SEN
```

其中 1.2.156.3088 为智能体身份码前缀，1.34C2.478BDF.3GF546.1.0SEN 为智能体身份码内容。说明：

**身份码前缀**
- （1）顶级弧：1表示ISO。
- （2）次级弧：2表示国家成员体。
- （3）第3级：156表示中国。
- （4）第4级：由国家OID注册中心批准的智能体互联专属节点。

**身份码内容**
- （5）第5级：智能体注册服务商序号（取值范围1～ZZZZZZ）。1表示一个智能体注册服务商A。
- （6）第6级：智能体供应商序号（取值范围1～ZZZZZZ）。34C2表示智能体注册服务商A分配给智能体供应商B的标识。
- （7）第7级：智能体本体序列号（取值范围1～ZZZZZZZZZ）。478BDF表示智能体注册服务商A分配给智能体供应商B的一个智能体本体序列号标识。
- （8）第8级：智能体实体序列号（取值范围0～ZZZZZZZZZ）。3GF546表示由智能体本体序列号为478BDF创建的智能体实体序列号标识。当注册对象为智能体本体时，其智能体实体序列号以特定字符0标识。
- （9）第9级：智能体身份码版本号（取值范围1~Z）。1表示智能体身份码版本为1。
- （10）第10级:校验码(取值范围0000～1EKF)。示例中的校验码为 0SEN，校验码的生成和验证方式在第四节中进行说明。

# 4. 校验码计算和使用

## 4.1 校验码的计算方式

智能体身份码的校验码（第10级）由前面的数据码（第1～9级）的内容计算得来(数据码和校验码均不包含第9级和第10级之间的分隔符`.`)，对于：

1.2.156.3088.1.34C2.478BDF.3GF546.1

其校验码采用 [AUTOSAR CRC-16/CCITT-FALSE](https://www.autosar.org/fileadmin/standards/R23-11/CP/AUTOSAR_CP_SWS_CRCLibrary.pdf) 方式计算，参数如下：生成多项式为 0x1021，初始值为 0xFFFF，不反射输入输出，最终不异或（xorout=0x0000）。计算过程如下：

（1）首先将字母统一转换为大写，再按 ASCII/UTF-8 编码转换为字节序列，作为 `CRC` 的输入数据。
比如：输入数据为字符串`1.2.156.3088.1.34C2.478BDF.3GF546.1`的 ASCII 字节流。（注意：此时`.`也属于输入字节）

（2）在进行 CRC 计算前，引入**盐值（Salt）**。盐值为各智能体注册服务商内部自行维护的固定字节序列，长度不少于 2 字节。将其拼接到（1）得到的 ASCII 字节流之后，形成最终的 CRC 输入字节序列。
比如：取 2 字节盐值 SALT = 0x1234（按字节序列为 0x12, 0x34），则最终输入为：
ASCII("1.2.156.3088.1.34C2.478BDF.3GF546.1") + [0x12, 0x34]

（3）初始化 16 位寄存器 `CRC` 为 0xFFFF。

（4）按字节依次处理输入数据。对每个字节 `B`，执行：
`CRC = CRC XOR (B << 8)`（即 将该字节异或到 `CRC` 的高 8 位）。
比如：字母 “B” 的 ASCII 码为 0x42（十进制 66，二进制为 0100 0010B）。在本算法中需将该字节左移 8 位后与 CRC 异或，即：`B << 8 = 0x42 << 8 = 0x4200`。由于当前 `CRC` 初始值为 0xFFFF，执行异或得到：
`CRC = 0xFFFF XOR 0x4200 = 0xBDFF`。其中 0xFFFF 的二进制为 1111 1111 1111 1111B，0x4200 的二进制为 0100 0010 0000 0000B，异或后得到 0xBDFF（1011 1101 1111 1111B）。

（5）对每个字节执行 8 次按位迭代（对应该字节的 8 个比特），每次迭代按如下规则更新 `CRC`：

- 若 `CRC` 的最高位（0x8000）为 1，则 `CRC = (CRC << 1) XOR 0x1021`；

- 否则 `CRC = (CRC << 1)`；并始终保持 `CRC` 为 16 位（超出部分丢弃，仅保留低 16 位）。

（6）全部字节处理完成后，得到最终 16 位 `CRC` 值，即为**校验码原值**。该值的取值范围为 0x0000 ~ 0xFFFF。
比如：对 1.2.156.3088.1.34C2.478BDF.3GF546.1 且设盐值 `SALT = 0x1234`计算得到 `CRC`校验码原值为 0x8FCF。

（7）将校验码原值采用 Base36 编码规则 进行转换：字符集为 0-9、A-Z（共 36 个字符，0 对应 0，9 对应 9，10 对应 A，35 对应 Z），转换后格式化为固定 4 位 Base36 字符串，不足 4 位时在左侧补 0。此 4 位 Base36 字符串即为校验码（第10级）。
比如：校验码原值 0x8FCF 经过 Base36 编码转换后得到 0SEN，则最终校验码为 0SEN。

（8）将校验码作为第 10 级追加到数据码后，并添加分隔符`.`，形成完整身份码。
比如：完整身份码为 1.2.156.3088.1.34C2.478BDF.3GF546.1.0SEN

## 4.2 校验码的使用

可按如下方式验证智能体身份码是否有效：

（1）将身份码分为数据码与校验码两部分：数据码为前 1～9 级，校验码为第 10 级（忽略第9级和第10级之间的分隔符`.`）。
比如：将 1.2.156.3088.1.34C2.478bDF.3GF546.1.0SEN 分为：
数据 1.2.156.3088.1.34C2.478bDF.3GF546.1 与校验码 0SEN。

（2）对数据码做与计算时相同的规范化处理，将字母统一转为大写。
比如：数据码规范化得到 1.2.156.3088.1.34C2.478BDF.3GF546.1。

（3）按与计算时相同的方式，在规范化后的 ASCII 字节流末尾拼接该 ARSP 对应的盐值字节序列，再按相同 CRC 参数计算校验码原值。
比如：重新计算得到校验码原值为 0x8FCF。

（4）将重新计算得到的校验码原值按相同的 Base36 编码规则转换为 4 位固定长度的 Base36 字符串（不足补 0）。
比如：校验码原值 0x8FCF → Base36 转换后得到 0SEN。

（5）将重新计算得到的校验码与身份码中携带的校验码（第10级）进行比较。若两者一致，则验证成功；否则身份码无效。
比如：重新计算为 0SEN，与携带的 0SEN 一致，则身份码有效。
